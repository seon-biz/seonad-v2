## 클로드 코드 사용 팁

### 1. 파일 첨부 순서
Claude Code UI 기준으로는 파일을 드래그 앤 드롭으로 첨부하면 됩니다.
- 맥락 이해가 중요하므로 **지식파일(K1~K4)을 먼저, 기획 MD(01~03)를 나중에** 붙이세요.
- 첨부 후 위 프롬프트를 입력합니다.

### 2. 한 번에 다 하지 말 것
프롬프트 마지막에 "각 단계마다 먼저 작업 내용을 요약하고, 내가 확인한 뒤 다음 단계로 진행해주세요"라고 적혀 있습니다.
→ 이게 중요합니다. 클로드 코드가 **한 번에 9개 페이지를 전부 만들려고 하면 오류 확률이 올라갑니다**.
→ 단계마다 결과를 확인하고, 이상하면 "이 부분만 다시 해줘" 지시.

### 3. 문제 생길 때 대처
- "01-foundation.md에 정의된 토큰을 Tailwind에 주입할 때 누락된 게 있어" 같은 구체적 지시 가능
- "이 섹션의 카피가 02-main-page.md 섹션 5와 다른데 다시 맞춰줘" 같은 대조 지시도 가능
- 크게 망했으면 "프로젝트 초기화하고 다시 시작해줘" 가능

### 4. 1단계 완료 후
클로드 코드가 메인 + 커리큘럼 페이지를 만들고 **대표님이 눈으로 확인한 상태**가 됐다면,
이 대화로 돌아와 "1단계 결과 확인 완료. 04-instructor.md 작성 이어서 진행해줘" 라고 알려주세요.

그러면 저는 04~09 파일을 이어서 작성하고,
9개 파일이 전부 완성되면 **2단계 바이브 코딩 프롬프트(= 나머지 페이지 추가용)** 를 별도로 작성해드립니다.
